Wizard segments — which to keep?

Multi-select. Each segment is a separate flow launched from the hub. Greyed-out items below are my recommended cuts for v1.

★

Hero "next action" CTA

The dynamic recommended next step. State-aware: "do gap report" → "run first mock" → "review feedback" → "do study item X" → "run second mock". Drives most engagement. Must keep.

1

Gap report flow

Bar list of weak dimensions with one-sentence "why" each (from matching graph + LLM tree-gen output). Click into a dimension to see worked example + linked study resources. Recommend keep.

2

Adaptive mock interview

The voice mock itself. Pre-flight screen (mic check + tips), live transcript view with the agent's adaptive question tree visualized, post-call processing wait, then feedback. Must keep — this is the product.

3

Mock feedback / score breakdown

Per-rubric-dimension score (4 dimensions from Phase 0 template), example moments from the transcript that scored well/poorly, and the auto-generated study items. Recommend keep.

4

Study plan

Curated list (from RESOURCES dict, Phase 0) of articles / videos / book chapters per weak dimension. Mark-as-done state. Resume where you left off. Recommend keep.

5

Mock history / trend

Tile + drill-down: score over time, per-dimension trend, ability to replay transcripts. Recommend keep tile-level, defer drill-down.

6

About this stage (process info)

Static Meta-style "what to expect at this stage" content. Useful but it's a CMS feature — every customer would need to seed copy. Recommend cut for v1; rely on PCS's existing process pages.

7

Reschedule / message recruiter

Deep-links into PCS-native flows. Doable but every link is a place we have to keep in sync with PCS routing. Recommend cut for v1; the candidate is one back-button from PCS already.
