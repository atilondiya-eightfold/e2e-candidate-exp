1

Hub spine — wizard launches from tiles

Dashboard is home. Big "Start guided prep →" CTA pulls them into the wizard; tiles deep-link to gap report / past mocks / study-plan items. Coming back from any wizard step lands on the hub. Better for repeat visits across days; "I'm 3 of 7 study items in" feels like a game progress bar.

2

Wizard spine — hub is a sticky header

Wizard is the body; hub collapses to a step bar at the top showing the journey and a "jump to overview" link. The candidate is always one step away from finishing the loop. Better for single-session completion; first-time visit feels less like a CMS, more like an experience.

3

Adaptive — first visit = wizard, return visits = hub

Detect state on entry. If gap report + at-least-one mock + ≥1 study-plan progress event don't all exist, show wizard (Hybrid 2). Otherwise show hub (Hybrid 1). One mode per visit, no toggling. Most engineering, but the cleanest UX for both audiences.

My recommendation: option 3 (adaptive). First-time candidates have zero data and a hub would look empty — wizard gives them momentum. Returning candidates have history worth surfacing — hub honors that. The cost is roughly one extra route + an entry-state selector; not a lot.
