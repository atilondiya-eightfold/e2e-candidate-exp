What changes vs. before

Wins

  • No live UI to build — drop the mic visualizer, transcript pane, tree viz
  • Agent can be hosted on any meeting backend the candidate's browser already supports
  • Pre-flight tips give us a place to set expectations (headphones, think aloud) — sets up better scoring
  • The launch card is reusable for every mock attempt; only the meta changes

Things to nail down

  • Where does the agent actually join? LiveKit room URL (re-uses Phase 0/1) vs. a real Google Meet link (means agent has to dial in as a SIP participant — much more work)
  • How do we know the candidate finished? Today's post-call hook fires on call-end; that's still fine, just plumb it to bring them back to this page
  • Mid-call abandonment — what state do we show on the hub if they joined but never came back?
  • No in-app captioning means hearing-impaired candidates depend on the meeting client's captions — acceptable for v1?
